# GNGM 프로젝트 디자인 가이드라인

## 프로젝트 개요
**GNGM (가는김에)**는 지역 커뮤니티 기반의 배송/심부름 서비스 플랫폼으로, "따뜻한 신뢰의 커뮤니티"를 지향합니다.

## API 데이터 처리 규칙 (중요)

### DTO 패턴 강제 사용
- **모든 API 요청/응답은 반드시 DTO 클래스를 사용해야 함**
- `Map<String, dynamic>` 형태의 데이터 직접 사용 금지
- DTO는 `/lib/models/dto/` 디렉토리에 구성

### DTO 명명 규칙
```
요청 DTO: {기능}RequestDto (예: LoginRequestDto, CreateRequestDto)
응답 DTO: {기능}ResponseDto (예: AuthResponseDto, RequestListResponseDto)
```

### DTO 구조 예시
```dart
// 요청 DTO
class LoginRequestDto {
  final String email;
  final String password;
  
  Map<String, dynamic> toJson() => {...};
}

// 응답 DTO  
class AuthResponseDto {
  final String token;
  final User user;
  
  factory AuthResponseDto.fromJson(Map<String, dynamic> json) => ...;
}
```

### 환경 설정
- API Base URL은 환경변수 사용: `String.fromEnvironment('API_BASE_URL', defaultValue: 'http://localhost:3000/api/v1')`
- 하드코딩된 localhost 사용 금지

## 디자인 철학
### 핵심 가치
- **익숙하지만 특별한**: 기존 앱의 검증된 UX + 차별화된 감성
- **신뢰 기반**: 모든 디자인 결정은 사용자 신뢰 구축이 최우선
- **커뮤니티 중심**: 개인보다는 동네 공동체 느낌 강조

### 컬러 사용 원칙
- **모노크로매틱 시스템**: 코랄 핑크 계열로 통일된 브랜드 아이덴티티
- **따뜻하고 감성적**: 친근하면서도 세련된 감성적 톤
- **독창적 브랜딩**: 기존 앱들과 차별화된 고유한 색감
- **명확한 위계**: Primary > Primary Light > Primary Dark 순서로 중요도 표현
- **상태 구분**: 기능별 색상으로 사용자 경험 향상 (성공=그린, 경고=앰버, 에러=레드)

### 폰트 적용 전략
- **브랜드 아이덴티티**: 'GNGM' 로고, 메인 타이틀에 지마켓 산스 사용
- **가독성 우선**: 본문, 버튼, 메뉴는 Pretendard 사용  
- **모던한 포인트**: 서비스 제목, 가격 표시에 지마켓 산스 선택적 적용
- **일관성 유지**: 같은 성격의 텍스트는 동일한 폰트 적용

### 폰트 사용 원칙
- **모던한 감성**: 지마켓 산스로 트렌디하고 깔끔한 브랜드 느낌
- **명확한 위계**: 폰트 크기와 굵기로 정보 구조 표현
- **최적 가독성**: 모바일 환경에서 14px 이상 보장
- **한국적 감각**: K-브랜드 감성을 담은 독특한 타이포그래피

## 개발 시 주의사항

### 컴포넌트 재사용성 원칙 (중요)
- **3회 이상 사용 시 필수 공통화**: 동일한 컴포넌트/화면이 3군데 이상에서 사용되면 반드시 공통 컴포넌트로 분리
- **재사용 가능한 컴포넌트 설계**: 버튼, 카드, 입력 필드, 모달, 리스트 아이템 등 표준화
- **Props 인터페이스 명확화**: 컴포넌트 재사용을 위한 명확한 속성 정의
- **컴포넌트 라이브러리 구축**: `/lib/widgets/common/` 디렉토리에 공통 컴포넌트 관리

### 사용자 경험(UX) 정보 설계 원칙 (중요)
- **정보 축약 및 단계별 제공**: 한 화면에 너무 많은 정보를 제공하지 않음
- **사용자 혼란 방지**: 핵심 정보만 메인 화면에 표시, 상세 정보는 별도 화면에서 제공
- **단계별 내비게이션**: 사용자가 필요한 정보를 찾기 위해 자연스럽게 화면을 이동하도록 설계
- **프로그레시브 디스클로저**: 기본 정보 → 상세 정보 → 추가 액션 순서로 정보 공개

### 화면 구성 가이드라인
```
메인 화면: 핵심 기능 3-4개만
    ↓
카테고리 화면: 해당 분야 서비스 리스트
    ↓  
상세 화면: 구체적 정보 및 액션
    ↓
액션 화면: 최종 처리 (연락, 신청 등)
```

### 컴포넌트 설계
- **재사용성**: 3회 이상 사용 시 필수 공통화
- **일관성**: 간격, 색상, 크기 시스템 준수
- **성능**: 불필요한 애니메이션 지양
- **모듈화**: 단일 책임 원칙에 따른 컴포넌트 분리

## UI/UX 작성 원칙 (중요)

### 금지사항
- **이모지 남용 금지**: 과도한 이모지 사용으로 AI 개발처럼 보이게 하지 말 것
- **허위 통계 금지**: "20,000명 사용자", "매칭률 99%" 등 근거 없는 수치 표기 금지
- **과장된 문구 금지**: "세계 최고", "혁신적인" 등 검증되지 않은 수식어 사용 금지
- **의미 없는 배지**: "BEST", "HOT", "NEW" 등 남발하지 말 것

### 권장사항
- **실제 데이터 기반**: 실사용자 후기, 실제 통계만 표시
- **간결한 표현**: 핵심 정보만 명확하게 전달
- **검증된 문구**: 사실 기반의 설명 사용
- **필요한 곳에만 아이콘**: 의미가 있는 곳에만 사용

### 텍스트 톤앤매너
- **친근하지만 전문적**: "안녕하세요" 보다는 "도움이 필요하신가요?"
- **구체적 정보 제공**: "빠른 매칭" 보다는 "평균 15분 내 매칭"
- **신뢰감 있는 표현**: 과장하지 않는 정직한 커뮤니케이션

### 브랜딩 요소
- **로고**: 심플한 홈 아이콘 + GNGM 텍스트
- **마스코트**: 친근한 동네 캐릭터 (추후 고려)
- **슬로건**: "가는김에, 우리 함께"
