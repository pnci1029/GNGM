# GNGM 개발 워크플로우 및 공수 산정

## 프로젝트 개요
- **목표**: 지역 커뮤니티 기반 배송/심부름 서비스 플랫폼
- **개발 기간**: 약 16-20주 (4-5개월)
- **개발 인원**: 풀스택 개발자 1명 기준

- [ ] 코드 품질 도구 설정 (ESLint, Prettier)

### 1.2 외부 서비스 연동 준비 (3일)

- [ ] **AWS/NCP** 계정 및 인스턴스 설정

## Phase 2: 데이터베이스 설계 (1주)

### 2.1 ERD 설계 및 검토 (2일)

#### 주요 테이블 구조:
```sql
-- 사용자
users (
  id, email, name, phone, profile_image, 
  provider_type, provider_id, rating_avg, 
  trust_score, location_lat, location_lng,
  created_at, updated_at, deleted_at
)

-- 서비스 요청
service_requests (
  id, user_id, category_type, title, description,
  pickup_address, pickup_lat, pickup_lng,
  delivery_address, delivery_lat, delivery_lng,
  fee_amount, status, created_at, expires_at
)

-- 서비스 제공
service_offers (
  id, provider_id, request_id, message,
  offered_fee, status, created_at
)

-- 채팅
chat_rooms (
  id, request_id, requester_id, provider_id,
  last_message, last_message_at, created_at
)

chat_messages (
  id, room_id, sender_id, message, message_type,
  read_at, created_at
)

-- 평가/리뷰
reviews (
  id, request_id, reviewer_id, reviewee_id,
  rating, comment, created_at
)

-- 신고
reports (
  id, reporter_id, reported_id, report_type,
  reason, description, status, admin_memo
)
```

## 주요 리스크 및 추가 고려사항

### 기술적 리스크
- **네이버 지도 API**: 연동 복잡도 높을 경우 카카오 지도로 대체 (+2-3일)
- **실시간 채팅**: Socket.IO 안정성 이슈 시 대안 검토 필요
- **위치 기반 서비스**: GPS 정확도 및 배터리 소모 최적화

### 외부 의존성
- **OAuth2 승인**: 카카오/구글 앱 심사 기간 (1-2주)
- **앱스토어 출시**: iOS/Android 스토어 심사 (1-2주)

### 법적/정책 고려사항
- **개인정보보호**: 위치정보, 채팅 데이터 처리 방침 수립
- **이용약관**: 서비스 이용약관, 개인정보 처리방침 작성
- **분쟁조정**: 사용자 간 분쟁 해결 프로세스 정의

### 운영 준비사항
- **고객지원**: FAQ, 1:1 문의 시스템
- **마케팅**: 초기 사용자 확보 전략
- **수익화**: 중장기 수수료 정책 수립
